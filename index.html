About the Project

A full-stack, LAN-based IT helpdesk platform built for internal use in factories and offices. The system allows employees to raise IT tickets without any login, while engineers manage tickets, assets, and spare parts through a modern admin dashboard. Designed to run completely offline on a Windows server using a lightweight Python/Django stack.

Key Features

User Portal (No Login Required)

  • Raise IT tickets for hardware, software, network & access issues
  • Track ticket status by Ticket ID — no account needed
  • Searchable Knowledge Base (FAQ) for self-service support

Engineer & Admin Dashboard

  • Live dashboard with real-time ticket counts & status breakdown
  • Full ticket lifecycle: Pending → Open → In Progress → Resolved → Closed/Rejected
  • Excel & PDF export for reports

Asset Management

  • Asset registry — category, location, AMC, invoice, financial year
  • Smart Excel import with auto header mapping
  • Bulk operations, Excel & PDF export

Spare Parts Management

  • Stock register with PO, warranty, quantity & pricing
  • Auto status: In Stock / Low Stock / Out of Stock / Issued
  • Spare-usage logs linked directly to tickets

Knowledge Base & SOP

  • FAQ management for self-service resolution
  • SLA & SOP document uploads for internal reference
  • Searchable manuals & procedure guides

Email Notifications

  • Configurable SMTP — Gmail, Office365 or corporate server
  • Emails on ticket creation, acceptance, resolution & closure
  • Automated routing to the right support team

Role & Delivery

End-to-end design and development — from initial requirements gathering and system architecture to backend engineering, frontend implementation, and final deployment on the client's Windows server. A one-person full-stack delivery from zero to production.

Requirements Analysis System Architecture Backend Engineering Frontend Development Database Design Deployment & Configuration
Deployment Approach

The system runs entirely on a local Windows PC or server — no internet required. A single start_server.bat script handles virtualenv setup, database migrations, static file collection, and Waitress WSGI server startup. Engineers and employees access the platform through any browser on the factory LAN. WhiteNoise serves static assets; python-dotenv manages environment configuration securely.
